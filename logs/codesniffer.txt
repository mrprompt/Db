
----------------------------------------------------------------
| ZEND CODING STANDARD: OPENING BRACE IN FUNCTION DECLARATIONS |
----------------------------------------------------------------

Function declarations follow the "BSD/Allman style". The function brace is on the line following the
function declaration and is indented to the same column as the start of the function declaration.

----------------------------------------- CODE COMPARISON ------------------------------------------
| Valid: brace on next line                      | Invalid: brace on same line                     |
----------------------------------------------------------------------------------------------------
| function fooFunction($arg1, $arg2 = '')        | function fooFunction($arg1, $arg2 = '') {       |
| {                                              |     ...                                         |
|     ...                                        | }                                               |
| }                                              |                                                 |
----------------------------------------------------------------------------------------------------


---------------------------------------
| ZEND CODING STANDARD: PHP CODE TAGS |
---------------------------------------

Always use <?php ?> to delimit PHP code, not the <? ?> shorthand. This is the most portable way to
include PHP code on differing operating systems and setups.


----------------------------------------
| ZEND CODING STANDARD: FUNCTION CALLS |
----------------------------------------

Functions should be called with no spaces between the function name, the opening parenthesis, and
the first parameter; and no space between the last parameter, the closing parenthesis, and the
semicolon.

----------------------------------------- CODE COMPARISON ------------------------------------------
| Valid: spaces between parameters               | Invalid: additional spaces used                 |
----------------------------------------------------------------------------------------------------
| $var = foo($bar, $baz, $quux);                 | $var = foo ( $bar, $baz, $quux ) ;              |
----------------------------------------------------------------------------------------------------


-----------------------------------------------------------------
| ZEND CODING STANDARD: DEFAULT VALUES IN FUNCTION DECLARATIONS |
-----------------------------------------------------------------

Arguments with default values go at the end of the argument list.

----------------------------------------- CODE COMPARISON ------------------------------------------
| Valid: argument with default value at end of   | Invalid: argument with default value at start   |
| declaration                                    | of declaration                                  |
----------------------------------------------------------------------------------------------------
| function connect($dsn, $persistent = false)    | function connect($persistent = false, $dsn)     |
| {                                              | {                                               |
|     ...                                        |     ...                                         |
| }                                              | }                                               |
----------------------------------------------------------------------------------------------------


-------------------------------------
| ZEND CODING STANDARD: LINE LENGTH |
-------------------------------------

It is recommended to keep lines at approximately 80 characters long for better code readability.
